# NovaX Optimization Papers

This directory stores paper scans and system notes that should guide future
autoresearch hypotheses.

## Workflow

Before a new optimization batch:

1. Search recent primary sources, especially arXiv, conference papers, and
   official compiler/runtime documentation.
2. Add or update a dated scan in this directory.
3. Extract each source into:
   - what the paper learned,
   - what it implies for NovaX,
   - which benchmark cases it should affect,
   - what experiment would test the idea.
4. Keep pushing the resulting scan and experiment commits only to the NovaX
   repository remote: `https://github.com/SS-012/novax.git`.
